# Agentic Edge Intelligence for IoT Threat Detection

This repository now provides a reproducible Python pipeline for IoT threat detection using a
hybrid and multi-agent approach:

- Stage 1: Cuckoo filter catches known attack signatures.
- Stage 2: Isolation Forest flags unknown anomalies.
- Stage 3: Random Forest classifies traffic type.
- Multi-agent reasoning votes on edge-agent outputs before mitigation action.

The notebooks in `notebooks/` remain as research artifacts, while production code lives in `src/`.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Run end-to-end pipeline

```bash
python main.py --agents 3 --threshold 0.6 --sample-per-class 40 --seed 42
```

This command will:

1. Preprocess and split data.
2. Train Isolation Forest and Random Forest.
3. Build Cuckoo filter from exclusive attack signatures.
4. Run multi-agent simulation.
5. Save:
   - `results/multi_agent_simulation.csv`
   - `results/metrics.json`
   - processed arrays and model artifacts under `data/processed/` and `models/`.

## Validate results from raw outputs

`results/metrics.json` is generated from `results/multi_agent_simulation.csv` every run.
No hardcoded report numbers are used in this pipeline path.

## Run tests

```bash
pytest -q
```
